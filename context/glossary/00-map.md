# Glossary Map

This directory keeps vocabulary stable across the repository.

## How to use this directory

### If you need the core framework terms
Read:
1. `01-framework-glossary.md`

### If you are introducing or renaming terms
Read:
1. `context/glossary/02-naming-and-term-introduction.md`
2. `context/review/05-vocabulary-drift-checklist.md`

## Documents

- `01-framework-glossary.md`
  - defines the core terms used across the repository
- `02-naming-and-term-introduction.md`
  - defines naming conventions and the process for adding or changing terms

## Directory rules

- glossary entries should be short and operational
- one canonical term per concept per scope unless aliases are explicitly recorded
- update the glossary before spreading a new term through many docs
- link to review checklists when term drift is a risk
